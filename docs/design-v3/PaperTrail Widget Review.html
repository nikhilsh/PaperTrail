The verdict — practical, but only if it tells the truth

Widgets are the right bet for PaperTrail: this is an app you open four times a year, and the shelf is how it stays hired in between. But a warranty widget earns its slot with ambient reassurance, not manufactured urgency — right now every surface screams countdown at items 321 days from expiry.

1
Amber urgency at 321 days W1 · P0

"Closing soon" with amber dots for items ~11 months out. Amber is a state, not a decoration. Threshold: only windows ≤ 60 days appear; otherwise show the quiet state ("Nothing closing soon").

2
Three formats for one number W2 · P0

"321 days", "321d", and the app's own "10 mo". One formatter: ≥90d → months · 8–89d → days · ≤7d → amber "6 days" · last day → "today".

3
"Warranty…" truncated headline W3 · P0

The serif headline is a cut-off event type. Headline = item name; sub = the stake with year; footer = mono months-left.

4
40pt gold "321d" alarm W4 · P1

Reads like a missile countdown for a TV that's fine. This slot is the coverage ring from spec C1 — the one widget that's always true.

5
Large widget duplicates medium W5 · P1

Same two items, less info, no dates. Cut it — ship Small ×2 + Medium + lock screen.

Widget logic — one source of truth

Units≥ 90 days → "10 mo" · 8–89 → "34 days" · ≤ 7 → "6 days" amber · last day → "today". Same formatter as the app. Never "321d".
Closing soonReturn windows always qualify; warranties only ≤ 60 days from expiry; register nudges ≤ 30 days from purchase. Max 2 rows, nearest first.
Quiet state"Nothing closing soon." + next expiry + portfolio line with sage ALL COVERED pill. This is the widget's default life — design it as the hero, not the fallback.
Pill colorsage = fine · amber < 60d · terra < 7d / expired. Amber never appears above 60 days, anywhere.
TypeItem names serif · stakes SF · metadata/counters mono caps. Headline is always the item, never a truncated event type.
Deep linksRing → Coverage tab · row → item dossier · quick-shelve → scanner. Reload on data change + daily 09:00.
MotionNone. WidgetKit is static — ring renders pre-filled, no shimmer, no fake countdown ticks.
