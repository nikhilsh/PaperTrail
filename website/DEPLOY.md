# PaperTrail website

The PaperTrail landing + iOS OTA install page. Static site generated by
`build.mjs` and deployed to **Fyra** at **papertrail.kaopeh.com**.

## What it does
`build.mjs` (zero runtime deps — Node built-ins + `fetch`):
1. Pulls the latest GitHub release carrying `PaperTrail.ipa` (prefers the rolling
   `adhoc-latest` tag).
2. Generates `out/ios/manifest.plist` pointing at that IPA, and bakes the
   `itms-services://…url=https://papertrail.kaopeh.com/ios/manifest.plist`
   install link + version/size into `out/index.html`.
3. Copies `public/assets/*` and `.deploy.yaml` into `out/`.

Because the manifest is served from the Fyra domain (real TLS, no redirect), OTA
install works reliably — unlike the old GitHub Pages attempt (which 301'd to a
non-Pages host).

## Manual deploy
```sh
cd website
node build.mjs                 # writes out/
cd out && fyra push            # ships it (uses ~/.fyra/config.yaml auth)
```
`.deploy.yaml` already carries the slug (`papertrail`) + domain, so no
`fyra link` is needed.

## CI deploy
`.github/workflows/deploy-website.yml` redeploys automatically:
- on pushes to `master` that touch `website/**`, and
- after a successful `Build Ad Hoc IPA` run (so the OTA manifest/version refresh
  whenever a new IPA ships).

Requires repo secrets **`FYRA_TOKEN`** (and optional `FYRA_SERVER`).

## Regenerating app screenshots
`render.mjs` / `crop.mjs` use `puppeteer-core` (a dev dependency) to capture the
screenshots in `public/assets/`. Only needed when the app UI changes:
```sh
npm install && npm run render
```
